The feature
10-minute read
The case against the single draft

The draft you abandoned is the draft you might have wanted.

Most note tools treat a document as one authoritative version. Every revision overwrites the last. The you-from-a-month-ago — her drafts, her alternate phrasing, her tentative outline — is lost to the undo buffer, and the undo buffer has long since been cleared.

Yarrow proceeds differently. When you want to try a different take, you open a new direction. Your original sits exactly where it was. You can switch between them, compare prose-level, or fold one into the other. Nothing is ever lost.

Under the hood it is git. At the surface, there is no git anywhere.

No commits, no branches, no HEAD. Just your writing, paths you can fork and return to, connections you can draw between ideas, and a quiet sense that the work you did is still there.

The dossier
All 15 features

Everything new — and the mechanics underneath.

Seven additions in 2.0 · eight mechanics that have been there since the start
i.
Pinned notes new

Anchor a handful of always-visible notes at the top of the sidebar. Right-click → Pin to top.

ii.
Quick capture new

⌘⇧Space pops a small window; Ctrl+Enter appends it to the scratchpad with a timestamp.

iii.
Journal & templates new

A month-grid calendar with dots on days you wrote; an optional templates/daily.md rendered into each entry.

iv.
Transclusion new

![[note]], ![[note#heading]], or ![[note^block]] — embeds another note's content inline.

v.
Path diffing new

Compare any two paths prose-level. Export a path as a portable Markdown bundle — notes + attachments + README.

vi.
Rebuilt connections graph new

Structured radial layout with directional arrows, a legend, a nearby-vs-all filter, expand-to-fullscreen.

vii.
History modal new

Now/Then panes, difference-only toggle, human timeline grouped by Today / Yesterday / Earlier this week.

viii.
Branching paths

Open a new direction to try a different take. The original stays untouched. Switch back any time.

ix.
Silent auto-save

No save button, no panic. Every pause in your typing is a checkpoint you can scrub back to — even weeks ago.

x.
Typed connections

Link notes by meaning — supports, challenges, came from, open question. Reverse links are automatic.

xi.
Daily notes

One markdown file per day at notes/daily/. Always on main; never splits.

xii.
Command palette

⌘K to do anything; ⌘O to fuzzy-jump. The mouse is optional.

xiii.
Open questions

Write ?? followed by a question anywhere; it appears in a dedicated panel. Unresolved tensions stay visible.

xiv.
Plain markdown, forever

Your notes live as .md files in a folder you choose. Open them in any editor.

xv.
Sync anywhere git runs

Point Yarrow at any git remote — GitHub, Gitea, a bare server. Tokens never leave your machine.

§3
In situ
Three views

Six things we will not change.

i.
Nothing you write is lost.
Every change is a checkpoint. Abandoned paths are an archive, not the trash.
ii.
Plain text, forever.
Notes are .md files. Git is invisible infrastructure, not a UI.
iii.
Local-first.
Yarrow works fully offline. Sync to a remote you own, or don't.
iv.
The vocabulary is human.
No commits, branches, HEADs, or merges. Just paths, checkpoints, and directions.
v.
Complexity lives out of sight.
The app is a Rust binary under ~15 MB. No bundled Chromium.
vi.
No account, no telemetry.
Yarrow never phones home. We don't even know you're using it.
