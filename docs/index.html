§1
The feature
10-minute read
The case against the single draft

The draft you abandoned is the draft you might have wanted.

Most note tools treat a document as one authoritative version. Every revision overwrites the last. The you-from-a-month-ago — her drafts, her alternate phrasing, her tentative outline — is lost to the undo buffer, and the undo buffer has long since been cleared.

Yarrow proceeds differently. When you want to try a different take, you open a new direction. Your original sits exactly where it was. You can switch between them, compare prose-level, or fold one into the other. Nothing is ever lost.

Under the hood it is git. At the surface, there is no git anywhere.

No commits, no branches, no HEAD. Just your writing, paths you can fork and return to, connections you can draw between ideas, and a quiet sense that the work you did is still there.

§2
What's new in 2.2
latest release
Released 2026-04-26

A redesigned radial centre, a Compare modal that diffs two notes, and an opt-in workflow built for bakers.

2.2 is the biggest interaction-design release since 2.0 — the radial right-click menu got a configurable centre (the Still Point), the right-click action set was reorganised around Inserts… and Format… modals, and the standard linear right-click menu now mirrors them as proper desktop submenus. Compare can finally diff two individual notes, not just two paths. A new opt-in Cooking additions extra unlocks five baking kits, a recipe URL clipper, inline timers, and a cook mode for hands-free reading. None of that is on by default — bakers turn it on, everyone else doesn't see it.

a.
Still Point gestures
three slots, twelve actions

Settings → Gestures binds tap, press-and-hold, and double-tap on the radial's centre dot to any of twelve actions — command palette, focus mode, today's journal, outline, live preview, scratchpad, and more. Defaults: tap → palette · hold → Quick Hand (a 3×3 modal of common actions) · double-tap → focus mode. There's also an alternate long-press destination — Constellation — that floats eight petals around the click point if you prefer a cursor-near radial to a centred grid.

b.
Compare two notes
notes mode default · paths mode optional

The Compare modal opens in Notes mode by default. Pick two notes by title and you get a side-by-side diff in any of the existing views (Summary / Track / Split / Cards / Marginalia). The original Paths mode — diff every note across two paths — is still available behind a toggle in the modal header. Pre-fills the active note as the left side when opened from the toolbar.

c.
Live-preview split & outline panel
writing surface upgrades

Toggle live preview from the command palette and the editor splits in two — writing on the left, fully-rendered markdown on the right, updated on every keystroke. The same NoteReader powers full reading mode, so typography stays consistent. The right-rail also gains an Outline button that pops a click-to-jump tree of headings + top-level bullets for the active note.

d.
Cooking additions
opt-in extra · off by default

Settings → Writing → Writing extras → Cooking additions. Flip it on and the rest of the cooking surface becomes visible: five baking kits (Recipe Card, Bake Log, Holiday Plan, Sourdough Schedule, Family Cookbook), a recipe URL clipper that pulls schema.org JSON-LD from major food sites, inline timers ([label](timer:25m)) that turn into tappable pills with a corner countdown, cook mode (bigger rendered text + screen wake-lock for hands-free reading), and a smart shopping list that scans a recipe's ## Ingredients section into a workspace shopping note. Off it stays hidden — most users aren't here for baking.

e.
macOS polish pass
native-feeling on Tahoe / Sequoia / Sonoma

Yarrow now mounts a real macOS menu bar — File / Edit / View / Window with the right accelerators, predefined cut/copy/paste/undo handed off to AppKit responders. acceptFirstMouse means the first click on an unfocused window registers in the editor instead of being eaten. Native overlay scrollbars are back (the universal styled-scrollbar rule is gated to non-Mac). prefers-reduced-motion: reduce is universally respected. The recipe URL clipper now uses reqwest, which fixes a "decoding chunks" failure on Cloudflare-fronted sites that ureq couldn't handle.

f.
Bibliography & multi-note PDF
research workflow extends

⌘K → "Insert / refresh bibliography" walks the active note's wikilinks, finds paper-tagged targets, and writes an APA reference block in place. ⌘K → "Export this path as PDF" prints every note in the current path as a single paginated document with a cover page + TOC. Builds on 2.1's research workflow.

§3
The foundation
kits, research, clinical, folders
Brought in by 2.1, still the bread and butter

A library of kits, two specialised workflows, and a quieter approach to privacy.

2.2's interaction-design changes sit on top of the workflow surface 2.1 introduced — the kit library, the research flow with paper-cards + @cite + BibTeX import, the privacy-first clinical surface with on-device PHI scanning + air-gapped sync, and optional sidebar folders. None of it is on by default — you opt in at the moment you want it.

b.
A research workflow
paper-card · @cite · BibTeX

Three pieces of one motion. The paper-card kit gives every paper a structured home. The @cite picker turns @bow into a wikilink to your Bowlby card. A .bib file from Zotero or BibDesk lands as one paper-card per entry — tagged, ready to cite.

1
Read & record
Drop a .bib file into Settings → Workspace → Import notes. Each entry becomes a paper-card with title, authors, year, journal, DOI, citation key.
2
Critique & connect
Open a paper-card. The kit's structure pre-fills the methodology, key findings, my critique, open questions, connections sections. Wikilinks tie cards together.
3
Cite & write
In any note, type @bow. The picker scopes to paper-tagged notes; Enter inserts a [[Bowlby 1969 — Attachment and Loss]] wikilink. The graph + reciprocal-link machinery picks it up like any other reference.
c.
A privacy-first clinical surface
never leaves your machine

Five clinical kits — SOAP, BIRP, DAP, supervision prep, reflective practice — plus intake, safety plan, treatment plan, risk assessment, termination, group session, behavioral chain, distortion log, and case formulation. All of them ship with the privacy gate already on: private: true plus the #clinical tag, and the file is registered in .git/info/exclude so it never enters a commit, let alone a sync.

on disk
Stays at notes/<slug>.md as plain markdown. You can edit it, back it up, read it in any editor — git just doesn't see it.
in the sidebar
Clinical-tagged tags render in rose. The note row shows a warm-amber ⊘ badge. Excerpt is redacted in every list view that shows note bodies.
while you write
A pure-JavaScript PHI scanner highlights SSN-, DOB-, phone-, email-, address-, and MRN-shaped patterns inline as you type. No IPC, no upload, no uploaded model — runs entirely in the editor.
in the welcome banner
"Where you left off" never bookmarks private or encrypted notes — the snippet that prompted this hardening is now structurally impossible to capture.
in search
The full-text search index excludes private bodies entirely; the command palette can't surface PHI snippets in results.
d.
Folders, optionally
if you want them

Right-click any note → "Move to folder…" — your notes group into collapsible sections at the top of the sidebar. The file always stays at notes/<slug>.md; the folder is pure sidebar metadata. A workspace with no folders defined renders identically to v2.0. Folders are created the moment you first move a note into one; "+ Create new folder" lives inline in the picker.

e.
Small but felt
the polish layer
  • Sharper text. Webfonts now load in parallel with CSS parsing (was an @import); the editor no longer applies a filter: blur(0.5px) on every note switch; zoom: 1 stops creating a stacking context at the default UI scale.
  • Wikilink autocomplete that ranks. Recently-opened notes float to the top; title-position matches outrank substring matches. Typing [[k gets you Knowledge before "Stuck on K".
  • Theme-aware graphs. The connection graph and tag graph repaint live when you flip light↔dark, instead of waiting for a panel reopen.
  • Differential spell-check. The plugin only rebuilds decorations on changed lines — measurable frame-time win on long notes.
  • IME composition guard. Pressing Tab while composing CJK candidates passes through to the input method instead of jumping the cursor to the next table cell.
  • External URLs open in your browser. An <a href="https://…"> click in the reader can no longer trap the Tauri webview — every external link routes through the OS default browser, with a fallback modal for the rare case where that fails.
  • Sync errors get categorised. Toasts now begin with "authentication failed:" or "network unavailable:" rather than the generic "sync failed" — you know whether to reconnect or check Wi-Fi.

The dossier
Everything since the start

Mechanics that have been here since v1 — and the new ones that joined them.

recent additions surface a "new" tag · earlier mechanics keep doing what they were quietly doing
i.
Pinned notes

Anchor a handful of always-visible notes at the top of the sidebar. Right-click → Pin to top.

ii.
Quick capture

⌘⇧Space pops a small window; Ctrl+Enter appends it to the scratchpad with a timestamp.

iii.
Journal & templates

A month-grid calendar with dots on days you wrote; an optional templates/daily.md rendered into each entry.

iv.
Transclusion

![[note]], ![[note#heading]], or ![[note^block]] — embeds another note's content inline.

v.
Path diffing

Compare any two paths prose-level. Export a path as a portable Markdown bundle — notes + attachments + README.

vi.
Rebuilt connections graph

Structured radial layout with directional arrows, a legend, a nearby-vs-all filter, expand-to-fullscreen.

vii.
History modal

Now/Then panes, difference-only toggle, human timeline grouped by Today / Yesterday / Earlier this week.

viii.
Branching paths

Open a new direction to try a different take. The original stays untouched. Switch back any time.

ix.
Silent auto-save

No save button, no panic. Every pause in your typing is a checkpoint you can scrub back to — even weeks ago.

x.
Typed connections

Link notes by meaning — supports, challenges, came from, open question. Reverse links are automatic.

xi.
Daily notes

One markdown file per day at notes/daily/. Always on main; never splits.

xii.
Command palette

⌘K to do anything; ⌘O to fuzzy-jump. The mouse is optional.

xiii.
Open questions

Write ?? followed by a question anywhere; it appears in a dedicated panel. Unresolved tensions stay visible.

xiv.
Plain markdown, forever

Your notes live as .md files in a folder you choose. Open them in any editor.

xv.
Sync anywhere git runs

Point Yarrow at any git remote — GitHub, Gitea, a bare server. Tokens never leave your machine.

§5
In situ
Three views

Six things we will not change.

i.
Nothing you write is lost.
Every change is a checkpoint. Abandoned paths are an archive, not the trash.
ii.
Plain text, forever.
Notes are .md files. Git is invisible infrastructure, not a UI.
iii.
Local-first.
Yarrow works fully offline. Sync to a remote you own, or don't.
iv.
The vocabulary is human.
No commits, branches, HEADs, or merges. Just paths, checkpoints, and directions.
v.
Complexity lives out of sight.
The app is a Rust binary under ~15 MB. No bundled Chromium.
vi.
No account, no telemetry.
Yarrow never phones home. We don't even know you're using it.
