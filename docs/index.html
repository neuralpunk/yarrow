02 · Why Yarrow exists
A note from the author Notes used to be static. Rigid. Something you preserved, not something you worked with. Yarrow is a notebook that thinks alongside you.

Thinking is not a straight line. You circle a problem, try a version, cross it out, come back to the first draft three days later and realise you were right the first time.

Yet every note app treats writing as a single, authoritative version. The draft is gone. The angle you didn't take is gone. The shape of how you got there: gone.

Yarrow is the opposite promise. Nothing you write is ever lost. Every save is a checkpoint. Every change of mind is a path you can return to.

Notes are no longer static. They work the way we think.
— On the 1.0 release
New in 1.0

The release where Yarrow stops being a tech demo, and starts being a notebook.

A daily-tool feature set — templates, tags, per-note encryption, a rebuilt Paths model, workspace switching — paired with a full editorial redesign that gives Yarrow the feel of a field notebook instead of a code editor.

01

A new visual language — Editorial.

Serif-forward typography on Source Serif 4, a paper-and-plum palette (ink-and-plum in dark), a slim 52px tool-rail, and a path ribbon at the top of every note. Writing in Yarrow should feel like writing in a notebook, not a dev tool.

02

Paths, rebuilt from scratch.

Paths are now collections of notes — named lenses over a single shared set of files — not git branches. One main per workspace, everything branches from it. Promote any side path to main and its era's peers come with it; the old main moves to the ghost zone, preserved as history.

03

Reading ↔ Writing mode.

A glasses/pencil toggle flips the editor between rendered prose and raw markdown. Headings, bold, inline code, and [[wikilinks]] collapse on every line you aren't editing.

04

Workspace modes.

Pick branch path mapping (default) for the full graph-and-paths experience, or basic notes for a plain notebook. Every mapped workspace anchors on a designated starting note — the ★ you always see in the sidebar.

05

Templates.

Six bundled — daily journal, meeting notes, book notes, vacation planner, project brief, morning pages — with placeholder substitution ({{date}}, {{weekday}}, {{cursor}}). New-note modal has a Start from chip row; edit them all in Settings → Templates.

06

Tags, first-class.

YAML frontmatter tags surface in a collapsible, counted sidebar panel. Click to filter; type #tag in the command palette to jump straight to the filtered view.

07

Local encryption — opt-in, per-note.

ChaCha20-Poly1305 with Argon2id key derivation and a 12-word BIP39 recovery phrase. Enable in Settings → Security; unlock once per session. Frontmatter stays plaintext so the graph, tags, and backlinks still work. ⌘L locks immediately. Everything happens on your machine — no network, no accounts.

08

Switch workspaces without losing your place.

A sidebar chip with a deterministic color badge per folder, a ⌘⇧O popover for recent workspaces, and a full remount on switch — so no state from the previous notebook leaks into the next.

09

Scratchpad, docked.

A resizable right-side pane (⌘⇧S), not a full-screen sheet. Editor right-click → Send to scratchpad. Keep-as-note from the footer with an optional came-from link to the note you were on.

10

Settings, built out.

Eight populated tabs — Appearance, Writing, Templates, Sync, Security, Workspace, Shortcuts, About — plus a search box across every row, and a shaped placeholder for Yarrow Sync.

11

Performance, made honest.

Initial JS payload dropped from ~985 KB → ~302 KB raw (322 KB → 92 KB gzipped) via lazy-loaded overlays and vendor chunk splitting. Hover-prefetch warms the next note into a 24-entry LRU. Reading ↔ Writing flips hot-swap via a CodeMirror Compartment — no flicker.

12

Quiet quality-of-life.

Three-state save indicator. Undo-delete toast. Word count + reading-time chip, with a selection count that swaps in live. Copy as markdown and Reveal in file manager on the right-click menu. A "today" chip on anything modified past local midnight. Static-site exports now ship with client-side search.

03 · The mental model

Three ideas, and the rest follows.

Yarrow's whole vocabulary fits on one line. Once you have these three, the rest of the app is just a place to do them.

I.

Notes

Plain markdown files. One per idea. Open, write, close. Yarrow saves silently while you type — there is no save button to miss.

Every save is a checkpoint you can scrub back to.

II.

Paths

Want to try a different take on a note — or on the whole workspace? Open a new direction. The old version stays safe.

Switch, compare prose-level, or bring two paths back together.

III.

Connections

Link notes by meaning: supports, challenges, came from, open question. Reverse links are added automatically.

The shape of your thinking becomes visible over time.

05 · Principles

The rules we wrote down, then didn't break.

  1. i.

    Nothing you write is ever lost.

    Every change is a checkpoint. Abandoned paths go into an archive, not the trash. If you wrote it, you can get back to it.

  2. ii.

    Plain text, forever.

    Notes are .md files on disk. Open them in any editor. Git is invisible infrastructure — not a UI you have to learn.

  3. iii.

    Local-first, by default.

    Yarrow works fully offline. Sync to a remote you own, on your schedule — or don't sync at all. Your notes stay yours.

  4. iv.

    The vocabulary is human.

    No commits, branches, HEADs, or merges. Just paths, checkpoints, directions, and connections. Words you'd use at dinner.

  5. v.

    Complexity lives in the backend.

    Yarrow is a Rust binary under ~15 MB. No bundled Chromium. Cold start in under a second on any machine from the last five years.

  6. vi.

    No account. No telemetry. No ads.

    Yarrow never phones home. We don't even know you're using it. That's the whole deal — and we don't intend to change it.

06 · FAQ

The things people ask first.

Do I need to know git to use Yarrow?

No. You never see git anywhere. The app uses git-the-library as the storage engine for checkpoints and paths, but the UI has no commits, branches, or HEAD — just your writing, paths, and connections. If you've never touched git in your life, Yarrow will feel like a normal note app that happens to remember everything.

Where do my notes live?

On your disk, in a folder you choose, as .md files. You can open them in VS Code, TextEdit, or any editor. You can back them up with any tool you already use. If Yarrow disappeared tomorrow, your notes would still be there, readable forever.

How is Yarrow different from Obsidian?

Obsidian is a rich editor that builds a graph from your links. Yarrow shares that premise, but adds two things Obsidian doesn't: time (every save is a checkpoint you can scrub back to) and alternate paths (try a different take without losing the first). If you want a linked-notes wiki, use Obsidian. If you want a notebook that remembers every draft, use Yarrow.

What happens when I open a new path?

A path is a way of saying "I want to try this differently." Every path branches from main — one trunk, many sibling branches. The old version stays exactly where it was; your main path is never overwritten. When you're done, you can switch back, compare the two, or promote a side path to become the new main.

What are "ghost paths"?

When you promote a side path to main, the old main and its siblings don't disappear — they move to a faded ghost zone to the left of the new main, with their original relationships preserved. Bring any one back to main and its era's peers come with it. Time moves forward; history is kept.

Is there sync? A mobile app? Collaboration?

Sync, yes — point Yarrow at any git remote (GitHub, Gitea, a bare server), hit one button in the toolbar. Mobile and real-time collaboration are not on the 1.0 roadmap. Yarrow is a single-user, local-first desktop app; we'd rather do one thing well than a lot of things adequately.

What does 1.0 mean? Is it really stable?

1.0 means we've passed the tech-demo phase. Saves are atomic. Concurrent-save races are resolved. There's still plenty we want to build — but we now stake the Yarrow name on data integrity. Nothing you write will be lost. That's the promise.

Is it really free?

Yes. Free as in beer, free as in speech. Yarrow is open source on GitHub under a permissive license. No paid tier. No usage limits. No "pro" version waiting in the wings.

How do I give feedback or report a bug?

Open an issue on the GitHub repo. We read everything. Feature requests that align with the principles above get priority.
